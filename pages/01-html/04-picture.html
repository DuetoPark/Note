🐸picture 태그 정의

클론 코딩을 하다가 picture 태그를 알게 되었다.

<picture> 태그
<picture> 태그는 웹 개발자에게 더 유연한 이미지 리소스를 제공한다.
가장 일반적으로 반응형 디자인 아트 디렉션에 사용된다. strong뷰포트의 너비가 변할때마다 하나의 이미지가 아닌, 여러개의 이미지로 브라우저의 뷰포트에 맞게 디자인할 수 있다.

⛄syntax

두개의 태그를 포함해야 한다.

  • source 태그
  • img 태그

❄️동작원리

브라우저는 미디어쿼리가 현재 뷰포트 너비와 일치하는 첫번째 source 요소를 찾은 다음 srcset 속성에 지정된 이미지를 표시한다.

img 요소는 source 태그의 마지막에 위치하고, fallback옵션으로서 source 태그에 매치되는 이미지가 없을 때 대체된다.

video와 audio 태그랑 유사하게 동작한다.

⚡예시코드

            
              < picture>
                < source media="(min-width: 650px)" srcset="baguette.jpg">
                < source media="(min-width: 465px)" srcset="cutting-baguette.jpg">
                < img src="img_girl.jpg">

🎉picture 태그가 좋은 이유

일단 뷰포트 크기마다 다른 이미지를 넣을 수 있다는 점이 진짜 매력적이다.
왜냐면 뷰포트 크기에 따른 이미지 변환 효과를 javascript의 흑마법 없이도 HTML 태그로 처리할 수 있기 때문이다.
HTML 상에 디자인만을 위한 div가 줄어드니까,
html의 코드가 보기 편하게 바뀌고 자바스크립트도 가벼워진다.
