🐣 논리 연산자(||, &&) 🐣

작성 날짜: 2021년 03월 19일

🐝 개요

다른 사람의 코드에서, '||'나 '&&'를 종종 보곤한다.
느낌상 뭔지는 알겠는데 정확하게 무슨 의미인지 물어보면 대답을 못하겠다.
그래서 정리해봤다.

🐞 OR 연산자 (||)

정의
const result = a || b;
OR 연산자는 인수 중 하나라도 true면 true를 반환한다.
다음의 예시를 보면, 피연산자가 모두 false인 경우를 제외하고 연산 결과는 항상 true이다.
예시 코드
                  
                    alert( true || true );   // true
                    alert( false || true );  // true
                    alert( true || false );  // true
                    alert( false || false ); // false
체이닝
OR 연산자를 여러 개 체이닝(chaining)하면 첫 번째 truthy를 반환한다. 피연산자에 truthy가 하나도 없다면 마지막 피연산자를 반환한다. 
                  
                    alert( undefined || null || 0 ); // 0 (모두 falsy이므로, 마지막 값을 반환함)
추가기능

추가기능 1 변수 또는 표현식으로 구성된 목록에서 첫 번째 truthy 얻기

변수에 값을 넣거나 빈칸으로 놔두고 싶은 순간이 있다. 이럴 때 유용하게 사용할 수 있다.
(false가 되는 값에는 false, undefined, "", null, Nan, 0이 있다.) 

                  
                    let volume = this.volume : "";

이벤트 리스너 안에 위의 예시를 넣었다고 가정하자. 만약에 해당 요소가 value속성을 가지고 있으면 변수에 값을 넣는다. 반대로 value속성이 없으면 변수는 빈칸이 된다.

추가기능 2 단락 평가(short circuit evaluation)

왼쪽부터 시작해서 오른쪽으로 평가를 진행한다. truthy를 만나면 나머지 값들은 건드리지 않은 채 평가를 멈춘다. 이런 프로세스를 '단락 평가'라고 한다. 

                  
                    true || alert("not printed");
                    false || alert("printed");

첫 번째 줄은 좌항이 truthy이므로 평가를 멈춘다. 그러므로 alert가 실행되지 않는다.
두 번째 줄은 좌항이 falsy이므로 alert가 실행된다.

🐟 AND 연산자 (&&)

정의
const result = a && b;
OR 연산자는 두 인수가 true여야지 true를 반환한다.
체이닝
OR 연산자를 여러 개 체이닝(chaining)하면 첫 번째 falsy를 반환한다. 피연산자에 falsy가 하나도 없다면 마지막 피연산자를 반환한다. 
                  
                    alert( 1 && 2 && null && 3 ); // null
                    alert( 1 && 2 && 3 ); // 마지막 값, 3
