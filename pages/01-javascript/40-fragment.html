작성 날짜: 2021년 03월 19일

🎽 개요

제로초님 게임 만들기 강좌에서 테이블 형성할 때 fragment에 대해 언급하셨다.
구글에 검색해보니까 다큐먼트 조각, 메모리, 성능 등의 단어가 많이 나왔다.
엄청나진 않지만 대단한 fragment에 대해 정리해보려 한다.

🎲 정의

documentFragment는 아주아주아주 작은 document 객체를 나타낸다. document의 경량화된 버전으로 사용된다.

활성화된 문서 트리 구조의 일부가 아니므로 fragment를 변경해도 문서에 영향을 미치지 않는다. 성능에도 영향이 없다!! reflow 개선에 효과적이다.

💎 사용법

fragment를 생성하고 그 안에다가 DOM 하위 트리를 9999999개 만든다. 그리고 appendChild나 insertBefore와 같은 메소드로 뭐가 가득 담긴 fragment를 DOM에 삽입한다.

모든 노드가 한 번에 문서에 삽입된다. 노드를 하나씩 삽입하면 노드 하나당 리플로우와 렌더링이 발생하지만 한꺼번에 삽입하면 리플로우와 렌더링이 단 한 번만 발생한다.

documentFragment는 document.createDocumentFragment() 메서드로 만든다.

예시 코드
            
              const table = document.querySelector("#table")
              const fragment = document.createDocumentFragment();

              for (let i = 0; i < 3; i++) {
                const tr = document.createElement('tr');
                for (let j = 0; j < 9999999; j++) {
                  const td = document.createElement('td');
                  tr.appendChild(td);
                }
                fragment.appendChild(tr);
              }
              table.appendChild(fragment); // 실제 노드에 있는 요소에 넣어버리기
