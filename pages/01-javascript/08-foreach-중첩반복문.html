🏊 forEach와 중첩반복문 🏊

🚴 개요

틱택토 게임만들기에서 게임이 끝나면 초기화를 해야한다.
반복문이 들어가는것도 알고 이차원 배열로 조작하는 것도 아는데 두개를 같이 어떻게 붙치지??
그 방법에 대해 알아보자.

🎳 forEach 정의

배열의 모든 요소에 콜백 함수를 적용할 때 사용하는 반복문이다.

🎾 forEach 문법

배열.forEach(콜백함수(요소, 인덱스, 배열) { 동작 }, thisArg);

문법을 목록으로 정리해봤다.

콜백함수
배열 안의 모든 요소에 적용될 함수를 의미한다.
요소
콜백함수를 처리할 배열의 요소를 의미한다.
함수 내에서 요소를 가르키는 매개변수가 된다.
생략할 수 없다.
인덱스
콜백함수를 처리할 배열 내 요소의 인덱스를 의미한다.
함수 내에서 요소의 인덱스를 가르키는 매개변수가 된다.
사용하지 않을 때에는 생략한다.
배열
forEach를 호출할 배열을 의미한다.
사용하지 않을 때에는 생략한다.
thisArg
콜백함수를 실행할 때 this로 사용하는 값을 의미한다.
사용하지 않을 때에는 생략한다.

⛳ forEach와 중첩반복문

이차원 배열은 반복문이 2개 필요하다.
반복문 안에 반복문이 들어간 중첩반복문을 사용한다.
아래는 예시코드이다. 

            
              칸들.forEach(function (줄들) {
                줄들.forEach(function (칸) {
                  // 동작
                });
              });

코드를 짜다보면, 마트료시카처럼 중중중중중첩반복문이 발생하는 경우가 있다.
이런 코드를 사람들은 좋지 않은 코드라고 말한다.

웬만하면, 중첩반복문은 최대한 두번 이내로 줄이는 습관을 들이자.

중첩반복문을 최대한 줄이면 좋은 코드가 된다.
하지만 아무리 줄여도 3개인 코드도 있다. 무조건 중첩반복문이 3개 이상이라고 해서 나쁜 코드가 되는 건 아니다.
