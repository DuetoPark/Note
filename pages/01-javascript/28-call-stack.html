💐 호출스택(Call stack) 💐

🌵 개요

자바스크립트에서 무조건 알아야 하는 스코프, 실행콘텍스트, 프로토타입, 비동기, 클로저중에 클로저를 공부해보자.

🌸 정의

호출 스택은 여러 함수들을 호출하는 스크립트에서 현재 어떤 함수가 동작하고 있는 지, 다음에 어떤 함수가 호출되어야 하는지를 제어하는 메커니즘이다.

Last In First Out(LIFO)형식이라는 것을 기억하자.

렉시컬 스코프에서는 함수의 '선언'이 중요했다면, 호출 스택에서는 '호출(실행)'이 중요하다.
뭐가 먼저 실행되고 뭐가 나중에 실행되는지 그 관계를 아는 것이 중요하다는 의미이다.

🌿 특징

  • 호출스택에 의해 함수가 끝나면, 함수가 사라지면서 함수 안의 변수가 날아가버린다. 하지만 이 문제는 스코프로 해결할 수 있다.
  • 스택은 유한하다. 그래서 할당된 공간보다 많은 공간을 차지하면 'stack overflow' 에러가 발생한다.
  • 스크립터의 함수 호출-호출 스택에 함수 추가-함수 실행의 과정으로 진행된다. 해당 함수에 의해 호출되는 모든 함수는 호출 스택에 추가되고 마지막 함수부터 차례로 실행한다.
  • 함수의 실행이 끝나면 호출 스택에서 빠져나간다. 넣기-쌓기-실행-빠짐의 반복이다. 개발자는 함수가 빠져 나감을 기준으로 함수의 실행 순서를 파악하면 된다.

🌋 호출 스택과 재귀 함수

재귀 함수를 실행하면 호출 스택이 터져버린다.
함수가 실행을 할수록 호출 스택에 쌓이기만 하기 때문이다.
사파리 외에는 대부분의 브라우저에서는 무조건 콜스텍이 터진다.
하지만, 비동기함수 특징을 이용한 꼼수를 쓰면 스택오버플로우를 해결할 수 있다. 

            
              function a() {
                  setTimeout(function() {
                      a();
                  }, 0);
              }

              a();

비동기인 setTimeout이 실행될 땐 어딘가에 함수를 저장하고 자기 할 일 끝났다며 나가버리는 특성이 있다. 기억해!

호출 스택에 다음과 같이 정렬된다.
a 함수-setTimeout
setTimeout이 어딘가에 함수를 저장하고 자기 할 일 끝났다며 나가버린다.그러면 a 함수도 실행을 마쳤으므로 호출 스택에서 나간다.

0밀리초뒤, 어딘가에 저장된 함수가 호출 스택이 들어온다.
호출 스택에 다음과 같이 정렬된다.
어딘가에 있던 setTimeout의 익명함수(콜백함수)-a-setTimeout
setTimeout이 실행될 땐 어딘가에 함수를 저장하고 자기 할 일 끝났다며 나가버린다.그러면 a도 실행을 마쳤으므로 호출스텍에서 나간다. 익명함수도 할 일 마쳤으니 호출스텍이서 나간다.

다시 0밀리초뒤, 어딘가에 저장된 함수가 호출 스택이 들어온다.
호출 스택에 다음과 같이 정렬된다.
어딘가에 있던 setTimeout의 익명함수(콜백함수)-a-setTimeout
setTimeout이 실행될 땐 어딘가에 함수를 저장하고 자기 할 일 끝났다며 나가버린다.그러면 a도 실행을 마쳤으므로 호출스텍에서 나간다. 익명함수도 할 일 마쳤으니 호출스텍이서 나간다.

이렇게 호출스택이 채워졌다 비워졌다를 반복하므로 콜스택 터지는 에러가 발생하지 않게된다.
