👀타이밍 이벤트👀

👸개요

나는 모달을 만드는 건 자신있다.
근데 효과 넣는 건 자신없다.
그래서 도전해봤다.
그러지 말았어야 했는데!

어쨌든 나는 도전했고, 타이밍 이벤트를 배웠다.

👮timing event

timing event 문법

window.setTimeout(function, milliseconds);

메소드의 첫번째 변수에 실행할 함수명을 넣거나,
함수를 작성한다.

메소드의 두번재 변수에는 지연할 만큼의 시간을 넣는다.
단위는 기입하지 않는다.

window없이 setTimeout 메소드 단독으로 나올 수 있다.

에잉, 이 함수 멈출래.
clearTimeout(function)
다른 버튼에 클릭 이벤트로 지정한다.

여러번 진행됬으면 좋겠다.
window.setInterval(function, milliseconds);

이것도 window없이 setInterval 메소드 단독으로 사용할 수 있다.
(기다리기)-(실행)-(기다리기)-(실행) 무한반복한다.

으악, 이 함수 멈출래.
window.clearInterval(function);
다른 버튼에 클릭 이벤트로 지정한다.

👼타이밍 이벤트 적용 안 할 때

용량이 큰 페이지에서 모달이 출력될 때,
애니메이션을 적용시키면 깜빡깜빡거리는 현상이 발생한다.
CSS 조작과 브라우저의 특징으로 인해 일어난다.
여기에서 말하는 브라우저의 특징이란, 코드가 작성된 순서대로 읽음을 의미한다.

  1. 애니메이션 실행 -(공백)- 클래스 추가
  2. 애니메이션 100%에서 {display: 1; opacity: 1;}
  3. (공백)에서 {display: none; opacity:1;}
  4. 클래스 추가 {display: 1; opacity: 1;}

공백일때 display가 none으로 변해서, 동작 도중에 깜빡깜빡하는 현상이 나타남.
setTimeout을 사용해서 애니메이션이 끝나기 전에 클래스를 추가하면 됨
