🏂 딕셔너리 자료구조

{key1:value1, key2:value2, ...}

객체를 다른 관점에서 보면, 일대일 매칭되는 사전식 자료구조라고 이해할 수 있다. 여기에서 일대일 매칭이란 중복되지않는 키와 그에 대한 값이 일대일로 연결된 형태를 의미한다.

왜 딕셔너리를 여기서 사용하냐면, 코드를 직관적으로 만들기 위함이다. 코드는 사람이 작성하고 사람이 읽는다. 그러므로 빠르고 쉽게 이해할 수 있도록 직관적이게 만들 필요가 있다.

과거의 내가 써놓은 코드를 보고, 이게 뭐지 한 적 있지않나??
아래의 코드처럼 변경한다면, 30년 후의 내가 봐도 이해할 수 있지 않을까? 

            
              let 헤더스타일 = {
                로고: {
                  width: "100%",
                  height: "50px",
                },
              }

              let 페이지헤더 = document.querySelector("#header");
              let 로고div = document.createElement("div");
              로고div.style.width = 헤더스타일.로고.width;
              로고div.style.height = 헤더스타일.로고.height;
              페이지헤더.appendChild(로고div);

예시가 적절하진 않지만 어쨌든 width를 박아버리는 것보다 이해하기 쉬워진다.

추가로, 딕셔너리를 쓰면 다국어 버전을 제공하는데 용이하다.
