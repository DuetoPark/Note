⛅ 개요

정말 어쩌다가 발견한 git gist.
생각보다 엄청나고 대단하다.
git gist의 사용법을 알아보자.

🎃 정의

코드를 에디터에서 작성한 것처럼 보여주는 도구

확장자명에 따라 코드의 하이라이트가 다르게 보인다. 또한 수정한 내용이 즉시 반영된다.

Github Gist에서 관리할 수 있다.

예시

🎄 사용 방법

  1. 1. 우층 상단 프로필 사진 옆의 '+' 아이콘을 누른다.

    깃허브에서 실행한다면 메뉴 중에 'New gist'를 클릭한다.
    깃허브 지스트에서는 '+'를 누르면 바로 코드 입력 화면이 나온다.

  2. 2. 코드 설명, 파일명, 코드를 입력한다.

    확장자를 꼭 입력해야한다. 확장자를 입력하지 않으면 하이라이트 효과가 활성화되지 않는다.

  3. 3. Create 버튼을 누른다.

    Create secret gist는 검색엔진이 탐색하지 못한다. 하지만 URL을 알면 누구나 볼 수 있다. secret 옵션이 기본 설정이다.
    Create public gist는 세상 모든 사람들이 볼 수 있다. 인터넷을 사용하는 강아지, 고양이, 외계인도 볼 수 있다.

  4. 4. 우측 상단 Embed의 주소를 복사, 원하는 페이지에 붙여넣기 한다.

    일반 페이지를 만들 때에는 주소 그대로 붙여넣기 한다.
    velog나 tistory 같은 블로그는 따옴표 안의 주소만을 복사 붙여넣기 한다.
    추가로, 편집(edit)과 삭제(delete) 버튼으로 gist를 관리할 수 있다. 버튼은 해당 gist에 들어가면 페이지 우측 상단 프로필 사진의 바로 아래에 있다.

👻 pre 태그와 비교

  1. 1. 스크린리더가 코드를 읽을 때, pre 태그는 문단처럼 읽고 Github Gist는 table 형식으로 읽어나간다.

    pre 태그는 코드를 줄 글 읽듯이 진행한다. 중간에 쉬지않고 처음부터 끝까지 쭉 진행되므로 중간에 놓치면 처음부터 다시 읽어야 한다.
    Github Gist는 코드 한 줄이 표의 행이된다. 특정 줄만 반복해서 읽을 수 있다.

  2. 2. pre 태그를 읽을 때 발음 정확도가 높다.

    input id="form-input"를 읽을 때, pre 태그는 '인풋 아이디 이콜 폼 인풋'으로 발음한다.
    Github gist는 '인푸티드 이콜 포민풋'으로 발음한다. 띄어쓰기와 '-'를 제껴버리는 느낌이다. 페이지 언어를 영어로 바꾸고 해당 스크립트 언어로 영어로 바꿔봤지만 개선되진 않았다.

  3. 3. Github Gist는 하이라이트 기능이 있다.

    속성이나 변수명등의 색상이 표시되므로 코드를 이해하는데 많은 도움이 된다. pre는 하이라이트 기능이 없다. HTML 태그 선언과 CSS 설정을 해야한다.
