📡 git bash 명령어 📡

⌛ 개요

매번 명령어를 찾아서 작성하는게 번거로와서 한 곳에 모아보았다.

📯 git 레포지토리 생성

깃허브 로그인
your repositories에서 new 버튼 클릭
레포지토리 이름 입력
레포지토리 주소 저장
레포지토리로 만들 폴더 우클릭
git bash 클릭

💸 git 로그인

다음의 명령어를 모두 입력한다. 1git config --global user.name "깃허브 user name"
2git config --global user.email "깃허브 이메일"

📪 로그인 확인

다음의 명령어를 입력한다.

git config --list
id(user name)와 email을 확인하는 명령어

🔫 깃 파일 생성

다음의 명령어를 입력한다. git init

💰 스테이징 전에 상황 확인하기

다음의 명령어를 입력한다.

git status
수정/신규 파일의 stage 상태를 출력한다.

🔨 스테이징

상황에 따라 필요한 명령어를 선택하여 입력한다.

1git add .
수정 또는 신규 파일 모두 스테이징하는 명령어
2git add 파일명
특정 파일만 스테이징하는 명령어
하나의 명령어로 여러개의 파일을 스테이징 할 수 있다.
공란으로 파일명을 구분한다.
3git add -p 파일명
수정 내역을 확인한 뒤에 원하는 부분만 스테이징하는 명령어
y는 yes, n는 no, q는 끝내기, s는 나누기를 의미한다.

🛀 커밋 메세지

다음의 명령어를 입력한다.

git commit -m "쓰고싶은 말"
작업 내용을 작성한다.
세부적으로 나눠서 작성한다.
명시적으로 작성한다

🔋 레포지토리 주소를 연결

다음의 명령어를 입력한다.

git remote add origin 레포지토리 주소
레포지토리 주소를 연결
git remote rm 지우려는 레포지토리 주소
레포지토리 주소 삭제
레포지토리 주소를 잘못 입력했을 경우에 사용

🔌 서버에 업로드

다음의 명령어를 입력한다.

git push -u origin 브랜치명(default: master)
스테이징한 파일들 서버에 업로드
웹 브라우저를 새로고침한다.
이 후에는 파일수정, git add, git status, git commit, git push를 반복한다.
