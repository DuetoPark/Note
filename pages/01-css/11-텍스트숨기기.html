🍕접근성을 고려한 텍스트 숨기기🍕

🍹개요

접근성과 최적화를 열심히 공부해두면 짱 멋있는 웹 프론트엔드 개발자가 될 것 같다는 느낌이 들었다. 그래서 접근성을 유지한 텍스트 숨김 방법을 공부해보기로 했다.

🍺그냥 숨기기

  1. 🍎display: none;

                      
                    display: none;

    display: none;은 정보를 표기하지 않겠다라는 의미이다.
    그러면 보조기기에 정보 전달을 하지 않음.
    즉, 내용을 숨겼다고 인식하는 게 아니라, 처음부터 없는 정보라고 인식.

  2. 🍏visibility: hidden;

                      
                    visibility: hidden;
                    position: absolute;

    display: none;와 동일하다.

  3. 🍊text-indent: -9999px;

                      
                    text-indent: -9999px;
    • 구글 검색엔진에서 스팸으로 인식
    • VoiceOver로 초점 이동시키면, 초점이 잡히는 문제 발생

  4. 🍋zero-pixel

                      
                    width: 0;
                    height: 0;
                    font-size: 0;

    width와 height를 0으로 지정할 때

    • NVDA와 VoiceOver는 contents를 읽음
    • VoiceOver로 초점 이동시키면, 초점이 잡히는 문제 발생

    font-size를 0으로 지정할 때

    • VoiceOver가 contents를 읽지 못함
    • 구글 검색엔진에서 스팸으로 인식

  5. 🍉opacity: 0;

                      
                    opacity: 0;
    • VoiceOver로 초점 이동시키면, 초점이 잡히는 문제 발생
    • 대화형 요소인 경우, 마우스가 반응함
    • VoiceOver가 읽지 못함

  6. 🍓position: absolute;

                      
                    position: absolute;
                    top: -9999px;
                    left: -9999px;
    • VoiceOver로 초점 이동시키면, 초점이 잡히는 문제 발생
    • 스팸으로 인식

🍸접근성을 고려한 숨기기

            
              .hidden{
                overflow: hidden;
                width: 1px;
                height: 1px;
                border: 0;
                margin: -1px;
                clip: rect(1px, 1px, 1px, 1px);
                clip-path: inset(50%);
              }
