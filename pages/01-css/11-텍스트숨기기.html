🍕 접근성을 고려한 텍스트 숨기기 🍕

🍹 개요

접근성과 최적화를 열심히 공부해두면 짱 멋있는 웹 프론트엔드 개발자가 될 것 같다는 느낌이 들었다.
그래서 접근성을 유지한 텍스트 숨김 방법을 공부해보기로 했다.

🍺 접근성을 고려하지 않은 방법

display: none;
display: none;은 정보를 표기하지 않겠다라는 의미이다.
그러면 보조기기에 정보 전달을 하지 않는다.
즉, 내용을 숨겼다고 인식하는 게 아니라, 처음부터 없는 정보라고 인식해서 보조기기와 스크린리더에 읽히지 않는다.
visibility: hidden;
display: none;와 동일하다.
text-indent: -9999px;
구글 검색엔진에서 스팸으로 인식한다.
VoiceOver로 초점 이동시키면, 초점이 잡히는 문제가 발생한다.
zero-pixel
제로픽셀 코드
                  
                    width: 0;
                    height: 0;
                    font-size: 0;
width와 height를 0으로 지정하면, NVDA와 VoiceOver는 contents를 읽을 수는 있다.
하지만, VoiceOver로 초점 이동시키면 초점이 잡히는 문제가 발생한다.
font-size를 0으로 지정하면, VoiceOver가 contents를 읽지 못하며 구글 검색엔진에서 스팸으로 인식한다.
opacity: 0;
VoiceOver로 초점 이동시키면, 초점이 잡히는 문제 발생한다.
대화형 요소인 경우, 마우스가 반응한다.
VoiceOver가 읽지 못한다.
position: absolute;
예시코드 
                  
                    position: absolute;
                    top: -9999px;
                    left: -9999px;
VoiceOver로 초점 이동시키면, 초점이 잡히는 문제가 발생한다.
구글 검색엔진에서 스팸으로 인식하다.

🍸 접근성을 고려한 방법

예시코드 

            
              .hidden{
                overflow: hidden;
                width: 1px;
                height: 1px;
                border: 0;
                margin: -1px;
                clip: rect(1px, 1px, 1px, 1px);
                clip-path: inset(50%);
              }
