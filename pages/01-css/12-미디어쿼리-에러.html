🌲 미디어쿼리 에러 🌲

작성 날짜: 2021년 03월 04일
수정 날짜: 2021년 03월 17일

🌾 개요

미디어쿼리를 하다보면 내가 정한 breakpoint가 깨지는 현상이 발생한다.
아니 640px로 설정했는데 왜 700px 전후에서 반응하는거야..?
이 거지같은 섬에 또 버려지지 말고 해결방법을 알아보자.

🌱 해결

결론부터 말하자면, 미디어쿼리의 시작점을 지정한 것처럼 끝점도 지정하면 된다.

그 이유는 CSS의 특징에 있다.
강조! 언어 자체가 직관적이기 때문에 미디어 쿼리 지정할 때는 시작점과 끝점을 정확하게 명시해야 한다.

당장 min-width를 넣어서 저 에러를 혼내주자.
하나의 CSS 파일에만 적용해도 다른 CSS파일의 break-point 문제는 해결된다. 코드는 다음과 같다. 

            
              @media screen and (max-width: 640px) and (min-width: 960px){
                /* CSS */
              }

              // 시작점: 640px, 끝점: 960px


              @media screen and (max-width: 960px) and (min-width: 1053px){
                /* CSS */
              }

              // 시작점: 960px, 끝점: 1053px
