🍕 접근성을 고려한 텍스트 숨기기 🍕

🍹 개요

접근성과 최적화를 열심히 공부해두면 짱 멋있는 웹 프론트엔드 개발자가 될 것 같다는 느낌이 들었다.
그래서 접근성을 유지한 텍스트 숨김 방법을 공부해보기로 했다.

🍺 접근성을 고려하지 않은 방법

display: none;
의미: 정보를 표기하지 않겠다.
보조기기에 정보 전달 차단
숨김이 아닌, 처음부터 없는 정보가 됨
visibility: hidden;
display: none;와 동일
text-indent: -9999px;
검색엔진이 스팸으로 인식
탭 이동할 때 포커스되는 문제 발생
zero-pixel
종류
  • width: 0;
  • height: 0;
  • font-size: 0;
width와 height가 0이면, NVDA와 VoiceOver는 정보를 인식 (VoiceOver에서 탭 이동할 때 포커스되는 문제 발생)
font-size가 0이면, VoiceOver가 정보를 인식 못 함 (구글 검색엔진에서 스팸으로 인식)
opacity: 0;
VoiceOver에서 정보를 인식 못 함
VoiceOver에서 탭 이동할 때 포커스되는 문제 발생
대화형 요소가 마우스에 반응
position: absolute;
VoiceOver에서 탭 이동할 때 포커스되는 문제 발생
검색엔진이 스팸으로 인식

🍸 접근성을 고려한 방법 

                
                  .hidden{
                    overflow: hidden;
                    width: 1px;
                    height: 1px;
                    border: 0;
                    margin: -1px;
                    clip: rect(1px, 1px, 1px, 1px);
                    clip-path: inset(50%);
                  }
