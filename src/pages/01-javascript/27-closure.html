🍧 클로저 🍧

🍲 개요

자바스크립트에서 무조건 알아야 하는 스코프, 실행콘텍스트, 프로토타입, 비동기, 클로저중에 클로저를 공부해보자.

👙 정의

클로저(Closure)
함수간의 어떠한 관계
클로저 관계: 함수와 함수가 접근할 수 있는 스코프
                  
                    var name = '나는 고구마';
      
                    function log() {
                      console.log(name); // 여기에 있는 name은 영원히 전역변수 name을 의미한다.
                    }
      
                    function wrapper() {
                      var name = '나는 오이'; // wrapper함수에서 선언한 name은 전역변수 name과는 완전히 다른 변수이다.
                      log();
                    }
                    wrapper(); // name = '나는 고구마';

log 함수와 wrapper 함수는 클로저 관계다.
왜냐하면, 무슨 코드를 쓰든 모든 함수들이 클로저 관계를 형성시키기 때문이다.

중요한 건 함수가 실행될 때 사용하는 변수값이 무엇인지 아는 것이다.

반복문과 비동기 함수가 만날 때 클로저 문제가 자주 발생한다.(for문 안에setTimeout함수 쓰는 것처럼)
여기에는 두 가지 이유가 있다.

  • 컴퓨터가 코드를 위에서 아래로 코드를 읽어나간다.
  • 컴퓨터가 동기는 읽고 비동기는 실행되기 전에는 넘겨버린다.

for문 안의 비동기 함수는 주석 처리한 것처럼 넘어가 버린다.
더불어, for문은 for문대로 돌아가니 i가 마지막 숫자를 가리킨다.
임무를 마친 컴퓨터는 다음 코드를 찾아 떠난다.

컴퓨터는 시킨대로 했고, 그 누구도 잘못하지 않았다. 하지만 원하는 동작을 구현하진 못한다. 왜냐하면 반복문 안의 비동기 함수를 실행했을 때 적용되는 i는 마지막 숫자만을 가리키기 때문이다.
여기에서 포인트는 비동기 함수 안의 변수는 '실행'될 때 값을 가져온다는 점이다.
그러므로 비동기함수가 '실행되는 순간에 변수가 뭔지' 찾아야 한다.

여기서 잠깐 표현을 다시 정정해야 한다.
"클로저 문제다. 클로저 문제가 발생했다."
이 표현들은 클로저의 특성을 사용해서 해결할 수 있는 문제라는 의미이다.

💣 예시 코드 

                    
                      for (var i=0; i<100; i++) {
                        setTimeout(function() {
                          console.log(i);
                        }, i * 1000);
                      }

위의 코드를 풀어서쓰면 다음과 같다.

                    
                      setTimeout(function() {
                        console.log(i);
                      }, 0 * 1000);
        
                      setTimeout(function() {
                        console.log(i);
                      }, 1 * 1000);
        
                      setTimeout(function() {
                        console.log(i);
                      }, 2 * 1000);
        
                      setTimeout(function() {
                        console.log(i);
                      }, 3 * 1000);
        
                      setTimeout(function() {
                        console.log(i);
                      }, 4 * 1000);
        
                      setTimeout(function() {
                        console.log(i);
                      }, 5 * 1000);
                      //--------------------------
                      setTimeout(function() {
                        console.log(i);
                      }, 99 * 1000);

setTimeout 내부의 함수에서는 i가 실행되지 않는다!!!!
setTimeout 콜백함수 내부의 i는 콜백함수 내부에서 i를 찾아야 한다.
콜백함수 내에 i가 없으니까 한단계 위로 i를 찾아나서야 하는데, 비동기함수이므로 실행되는 순간이 되야지 i가 뭔지 찾는다.
실행될 때의 시점에서는 i가 100인 상태라서 그 값을 이용할 뿐이다. 클로저 문제를 해결하려면 다음의 코드처럼 수정해야한다. 

                    
                      에러코드)
                      for (var i=0; i<100; i++) {
                        setTimeout(function() {
                          console.log(i);
                        }, i * 1000);
                      }
        
                      해결책)
                      // 함수를 실행하는 방식
                      for (var i=0; i<100; i++) {
                        function 클로저(j) {
                          setTimeout(function() {
                            console.log(j);
                          }, j * 1000);
                        }
                        클로저(i);
                      }
        
                      // 즉시 실행 함수를 사용하는 방식
                      for (var i=0; i<100; i++) {
                        (function 클로저(j) {
                          setTimeout(function() {
                            console.log(j);
                          }, j * 1000);
                        })(i); //
                      }

🔨 클로저 문제 해결법

  • 즉시 실행 함수
  • 매개변수
  • 렉시컬 스코프
                    
                      for (var i=0; i<100; i++) {
                        (function 클로저(j) {
                          setTimeout(function() {
                            console.log(j);
                          }, j * 1000);
                        })(i);
                      }

먼저, 즉시 실행 함수 인해 인자로 들어간 값이 매개 클로저 함수의 매개변수가 된다.

진행에 따라 i=1일 때 매개변수 j=1, i=2일 때 매개변수 j=2, ... 이렇게 j에 값이 저장된다.
비동기 함수가 '실행'되면 렉시컬 스코핑에 의해 변수를 찾아 저장된 값을 대입한다. 

                  
                    // i = 0, j --> 0
                    function 클로저(j) {
                      setTimeout(function() {
                        console.log(j); // 콜백함수 '실행'되는 순간, j=0 대입
                      }, 0 * 1000);
                    }
      
                    // i = 1, j --> 1
                    function 클로저(j) {
                      setTimeout(function() {
                        console.log(j); // 콜백함수 '실행'되는 순간, j=1 대입
                      }, 1 * 1000);
                    }
      
                    // i = 2, j --> 2
                    function 클로저(j) {// j는 3이다.(매개변수니까!)
                      setTimeout(function() {
                        console.log(j); // 콜백함수 '실행'되는 순간, j=2 대입
                      }, 2 * 1000);
                    }
      
                    // i = 3, j --> 3
                    function 클로저(j) {
                      setTimeout(function() {
                        console.log(j); // 콜백함수 '실행'되는 순간, j=3 대입
                      }, 3 * 1000);
                    }
      
                    //---------------------------
      
                    // i = 99, j --> 99
                    function 클로저(j) {
                      setTimeout(function() {
                        console.log(j); // 콜백함수 '실행'되는 순간, j=99 대입
                      }, 99 * 1000);
                    }

😚 즉시 실행 함수

함수 표현은 함수를 정의하고, 변수에 함수를 저장하고 실행하는 과정을 거친다.
하지만 즉시 실행 함수는 함수를 정의하고 바로 실행하여 '저장'과정을 거치지 않는다. 정의하자마자 냅다 호출하는것을 즉시 실행 함수라고 한다.
즉시 실행 함수의 형태는 다음과 같다. 

                    
                      (function () {
                        //동작
                      })();

두 개의 괄호는 각각 다른 의미를 가진다.
첫 번째 괄호는 익명함수를 둘러싼다. 이는 전역 스코프 변수로부터 즉시 실행 함수가 자유롭다는 것을 의미한다.
두 번째 괄호는 즉시 실행 함수를 생성하는 괄호이다.
