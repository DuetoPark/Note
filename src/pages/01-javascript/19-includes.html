🎡 includes 🎡

⛲ 개요

코드가 글 읽듯이 자연스럽게 읽히면 아주 잘 짠 코드라고 했다.
간결하고 중복이 없으면 아주 좋다고 했다.
하지만 내 코드는 지저분하구 길구 고소한 맛이 나구.
아직 내 코드는 마치 기철이가 와서 한 마디 할 것 같은 모습을 하고있다.

점점 길어지는 if문의 조건문을 보며, 착하긴 한데 쟤 좀 어떻게 하고싶다고 생각했다.
나는 그랬다.
우리 엄마가 날 보는 심정이었을까?
이런 생각을 해소시켜주는 메소드가 있다.
includes에 대해 알아보자.

🎠 정의

includes
배열에 특정 요소를 포함하는지 판별하는 메소드
특정 요소를 찾으면 true, 없으면 false를 반환
문자나 문자열을 비교할 때, 대소문자를 구분하니 주의해야 한다.
includes 문법
배열.includes(탐색할 요소, 인덱스);
인덱스: 검색을 시작할 위치
인덱스는 생략 가능하며 기본값이 0이다.

🎢 예시

if 문의 조건문이 길 때가 있다. 한 눈에 들어오지도 않고 너무나 위협적이다. 이럴 때 includes를 적용하면 된다.
if문의 조건문을 한 번 보자.

(점수표[내선택] - 점수표[컴퓨터의선택(이미지좌표)] === -1 || 점수표[내선택] - 점수표[컴퓨터의선택(이미지좌표)] === 2)

[-1, 2].includes(점수표[내선택] - 점수표[컴퓨터의선택(이미지좌표)])

조금 덜 공격적으로 변했다.
아직 조금 지저분한 상태로 완전히 깨끗한 상태로 바꾸면 다음과 같아진다. 

                  
                    let 내점수 = 점수표[내선택];
                    let 컴퓨터점수 = 점수표[컴퓨터의선택(이미지좌표)];
                    let 점수차 = 내점수 - 컴퓨터점수;
      
                    if ([-1, 2].includes(점수차)){
                      //동작
                    }

기억하자.
코드는 간결하고 한눈에 보기 쉬워야 한다.
오늘도 방망이 깎는 노인처럼 코드를 깎아내야겠다.
