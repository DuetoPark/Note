🐥 콜바이밸류(call by value) 🐥

🐢 개요

함수의 매개변수에 인자를 넣으면 어떤 일이 일어나는지 알아보자.

🐎 콜바이밸류(call by value)

call by value
함수의 인자와 매개변수(parameter) 간의 관계를 의미
인자: 함수 실행할 때 괄호 안에 선언한 값
매개변수: 함수 선언할 때 괄호 안에 넣은 변수
                      
                        function 함수 (매개변수) {
                          console.log(매개변수);
                        }
        
                        함수(3); // 인자: 3

함수를 실행하면서 인자에 3을 넣었다.

선언한 함수의 매개변수에 3을 대입한다.

var 매개변수 = 3;이므로 console.log는 3이 출력된다.

                      
                        function 함수(매개변수) {
                          매개변수 = 10;
                          console.log(매개변수);
                        }
        
                        var 인자 = 5;
                        함수(인자); // 10
                        console.log(인자); // 5

함수 내부에서 매개변수가 10으로 수정되므로 함수(인자)는 10을 출력한다.

매개변수는 함수 밖을 빠져나가지 못하기 때문에 console.log(인자)는 5를 출력한다. 

                      
                        function 함수(매개변수) {
                          매개변수.a = 10;
                          console.log(매개변수);
                        }
        
                        var 인자 = {a: 5};
                        함수(인자); // 10
                        console.log(인자); // 10

매개변수에 {a: 5}를 대입한다.

함수 내부에서 매개변수.a가 10으로 수정되므로 함수(인자)는 10을 출력한다.

여기에서 주의할 점은 매개함수에 객체를 대입한다는 점이다. 객체를 대입하면 참조관계가 된다. 그러므로 인자.a도 10이 된다.
console.log(인자)가 10을 출력하는 이유이다.

🐝 콜바이레퍼런스(call by reference)

call by reference
콜바이 레퍼런스는 자바스크립트에 존재하지 않는다!
자바스크립트에서 인자에 어떤 것을 넣든 모두 콜바이밸류이다.

객체의 속성을 수정하는 부분에서 참조 관계를 띌 뿐이다. '객체'를 원시값으로 수정할 시에는 참조 관계가 깨져버린다.
아래의 코드는 콜바이레퍼런스라는 주장의 반례이다. 

                  
                    function 함수(매개변수) {
                      매개변수 = "오이"; // 객체 --> 문자열로 수정, 참조 관계 깨짐
                      console.log(매개변수);
                    }
      
                    var 인자 = {a: 5}; //객체
                    함수(인자); // "오이"
                    console.log(인자); // {a: 5}
