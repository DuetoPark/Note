🎁 개요

처음 만들었던 내 소개 페이지에 슬라이더를 넣었었다. 그땐 라이브러리를 이용했었는데 이번엔 직접 구현을 해봤다.
사실은 오늘 본 강의 내용이다.
추가로 라이브러리의 종류와 사용법도 정리해보려고 한다.

👔 동작

'드래그'로 슬라이더를 넘긴다.

🃏 구조

  • slider-container
  • slider-items(원하는 만큼)

🚴 키워드

이벤트

  • mousedown
  • mouseup
  • mousemove
  • mouseleave

좌표

  • startX: 드래그 시작
  • scrollLeft: 스크롤 위치
  • lastX: 드래그 끝
  • walk: 마우스 이동거리

플래그

isDown: 마우스를 누르고 드래그 할 때에만 동작함

👑 구현

mousedown

  1. 1. 플래그를 true로 변경한다.
  2. 2. startX의 좌표를 구한다.

    startX = event.clientX - slider.offsetLeft
    slider.offsetLeft: slider-container의 x좌표 위치 slider-container의 margin 부분에서는 동작하지 않도록 하기 위함이다. (동작 범위: content부터 border까지)

  3. 3. scrollLeft의 좌표를 저장한다.

    scrollLeft = slider.scrollLeft;

mouseleave & mouseup

플래그를 flase로 바꾼다.
동작을 제한하기 위함이다.

mousemove

  1. 1. isDown이 false면 함수 동작을 멈춘다.

    if (!isDown) return;
    마우스에서 손을 떼거나 슬라이더 바깥으로 마우스 포인터가 나갔을 때 슬라이더의 동작을 멈춘다.

  2. 2. lastX 좌표를 구한다.

    lastX = event.clientX - slider.offsetLeft
    slider.offsetLeft: slider-container의 x좌표 위치 slider-container 내부에서의 실시간 위치를 의미한다.

  3. 3. walk를 구한다.

    walk = lastX - startX; 마우스를 누른 순간부터 현재 위치까지의 거리이다. 슬라이더를 휙휙 넘기거 싶다면 walk 값에 n배수 처리한다.((lastX - startX) * n)

  4. 4. slider.scrollLeft를 변경한다.

    slider.scrollLeft = scrollLeft - walk; scrollLeft: mousedown 이벤트에서 저장했던 slider의 스크롤 위치

😊 라이브러리

유명한 Slider 라이브러리는 다음과 같다.

  • Slick
  • Swiper
  • Flickity
  • Keen-Slider

👽 명명

라이브러리를 찾아보면서 알게 된 것인데, slider를 캐러셀(Carousel)이라 부른다. 더 다양한 슬라이더 라이브러리를 사용하고자 한다면, 구글에 'carousel library'라고 검색하면 된다.
