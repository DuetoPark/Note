📹 개요

localStorage에 데이터를 저장하는 코드를 연습해봤다.
혼자서 한 건 아니고 30일 챌린지 강의를 수강하며 만들었다.
문득 TO-DO-LIST 짤 때가 떠올랐다.
정말 하나도 몰랐어서 울면서 만들다가 치킨으로 위로받았었다.
이젠 localStorage도 뿌실 수 있다.

📀 예시

입력창에 값을 입력하고 엔더 혹은 '+Add Item' 버튼을 누른다.
페이지를 로드해도 데이터는 사라지지 않는다.

🔉 정의

브라우저의 Storage에 데이터를 저장하는 것을 의미한다.

페이지가 로드되어도 데이터는 날아가지 않는다.

저장한 자료는 페이지 트로토콜별로 구분한다. http와 https는 서로 다른 로컬스토리지에 저장한다.

'key' 값을 가지며 정수키는 자동으로 문자열로 변환한다.

📟 사용방법

데이터 저장
localStorage.setItem('key', '값');
키와 값을 입력한다.
값에는 숫자, 문자, 객체를 넣을 수 있다.
객체를 넣을 때에는 문자열로 바꿔주는 작업이 필요하다.(JSON.stringfy(객체))
데이터 가져오기
const variable = localStorage.getItem('key');
데이터 제거
localStorage.removeItem('key');
localStorage.clear();
removeItem은 특정 키 만을 제거한다.
clear는 로컬스토리지 항목의 전체를 제거한다.

📠 로컬스토리지 확인

로컬스토리지는 개발자도구의 'applicaton' 탭에서 확인할 수 있다.

💩 JSON 겉핥기

로컬스토리지에 객체를 저장하려면 JSON.stringify 과정을 거쳐야 한다.

일단 먼저 로컬 스토리지에 객체가 들어왔을 때 브라우저는 굉장히 난감해 한다. 객체의 우왕좌왕하다가 객체를 열지 못하고 글자인 "[Object Object]"로 바꾼다.

브라우저가 객체 안의 글자를 읽을 수 있게 객체를 JSON 문자열로 변환하는 작업이 필요하다. 이 작업을 JSON.stringify가 담당한다.

문자열로 변환된 객체를 원 상태로 되돌리는 명령어는 JSON.parse이다. JSON 문자열의 구문을 분석하고 ,그 결과에서 Javascript 값이나 객체를 생성한다.
