👀 타이밍 메소드 👀

👸 개요

함수 실행을 지연하거나 반복하는 메소드에 대해 알아보자.

👮 setTimeout

setTimeout
동작을 지연시키는 메소드
window 없이 함수 단독으로 사용 가능
window.setTimeout(콜백함수, milliseconds);
setTimeout의 콜백함수
동작할 함수
함수를 직접 입력하거나, 함수명을 입력한다.
setTimeout의 milliseconds
지연할 만큼의 시간
단위는 기입하지 않으며, 1000은 1초를 의미한다.

setTimeout은 CSS와 함께 사용하면 정확한 동작을 구현할 수 있다.
예를 들어, display와 opacity 속성을 같이 조작할 때 유용하다.
display는 trasition 속성이 적용되지 않는다. setTimeout를 통해 display 속성을 변경하는 동작과 opacity 속성을 변경하는 동작을 시간차를 두어 실행하면 원하는 동작이 이뤄진다.

🍤 setInterval

setInterval
함수를 무한반복 실행할 때 사용하는 메소드
window 없이 함수 단독으로 사용 가능
window.setInterval(function, milliseconds);
setInterval의 콜백함수
무한히 동작할 함수를 의미
함수를 직접 입력하거나, 함수명을 입력한다.
setInterval의 milliseconds
입력한 시간 간격으로 동작이 실행
생략하면 무한히 동작한다.

🍵 clearInterval

clearInterval
setInterval을 멈추는 메소드
window.clearInterval(setInterval함수);

🐴 clearTimeout

clearTimeout
setTimeout을 멈추는 메소드
window.clearTimeout(setTimeout 함수);

🐋 타이밍 메소드와 스코프

clearTimeout은 setTimeout과 스코프가 다르면 호출스텍 문제가 발생한다.

이를 해결하려면 전역 함수에 변수를 빼내야 한다. 그 변수에 setTimeout를 저장하면 호출스택 문제는 해결된다. 

                    
                      var 타임아웃 = "";
        
                      if (조건1) {
                        타임아웃 = setTimeout(function() {
                          // 콜백함수 동작
                        }, 2000);
                      } else if (조건2) {
                        clearTimeout(타임아웃);
                      }
