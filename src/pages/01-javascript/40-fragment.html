📝 fragment 📝

🎽 개요

제로초님 게임 만들기 강좌에서 테이블 형성할 때 fragment에 대해 언급하셨다.
구글에 검색해보니까 다큐먼트 조각, 메모리, 성능 등의 단어가 많이 나왔다.
엄청나진 않지만 대단한 fragment에 대해 정리해보려 한다.

🎲 정의

documentFragment()
document의 경량화된 버전
활성화된 문서 트리 구조의 일부가 아님
DOM 트리처럼 동작 (메모리 상에서만 존재하는 빈 문서 템플릿과 유사)
자식 노드를 메모리상에서 조작한 후, 라이브 DOM에 추가
인수로 전달하면, 자식 노드 구조 전체가 삽입(documentFragment 노드 자체는 무시)
reflow 개선에 효과적

💎 사용법

  1. fragment 생성
    메소드: createDocumentFragment
    예시: var frag = document.createDocumentFragment();
    메모리상에서 노드를 생성한 형태
    라이브 DOM에 삽입하는 것 보다 가볍고 빠름
  2. 자식요소 생성
    메소드: createElement
    예시: var div = document.createElement('div');
  3. fragment에 자식요소 삽입 생성
    메소드: appendChild
    예시: frag.appendChild(div);
  4. 라이브 DOM에 삽입
    메소드: appendChild
    예시: section.appendChild(frag);
    fragment의 자식 요소 개수에 상관없이 단 한번으로 라이브 DOM에 삽입
    fragment는 무시되고, 자식 노드만이 삽입
                  
                    const table = document.querySelector("#table")
                    const fragment = document.createDocumentFragment();
      
                    for (let i = 0; i < 3; i++) {
                      const tr = document.createElement('tr');
                      for (let j = 0; j < 9999999; j++) {
                        const td = document.createElement('td');
                        tr.appendChild(td);
                      }
                      fragment.appendChild(tr);
                    }
                    table.appendChild(fragment); // 실제 노드에 있는 요소에 넣어버리기
