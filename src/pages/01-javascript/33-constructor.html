☁️ 생성자(Constructor) ☁️

☔ 개요

생성자를 이전에 공부했던 주제다. 여기에는 새로 배운 내용 만을 정리하려고 한다.

⚡ 생성자

생성자에도 프로토타입을 넣어줄 수 있다.
선언하는 방법은 다음과 같다.

생성자명.prototype = prototype변수명;
__proto__를 사용하지 않는다.

❄️ 팩토리와 생성자의 차이

팩토리랑 다른점은, console.log 찍었을 때 생성자가 객체 앞에 적혀있다.
객체 앞에 생성자가 있으면 이 생성자로 인해 만들어진 객체구나 라고 이해하면 된다.

객체지향 프로그래밍에는 생성자를 사용하고,
객체지향 프로그래밍을 안 할 때에는 팩토리를 사용한다.

🌊 new를 선언하지 않으면 벌어지는 일 

                  
                    function 엉덩이(좌궁둥, 우궁둥) {
                      this.left = 좌궁둥;
                      this.right = 우궁둥;
                    }
      
                    var 오이 = new 엉덩이("조지", "마를린"); // 변수에 객체 저장
                    var 포도 = 엉덩이("클로이", "데이빗"); // 변수에 함수 저장
      
                    console.log(window.left); // 클로이
                    console.log(window.right); // 데이빗

new를 제대로 선언하면 변수에는 객체가 저장된다.

console로 찍으면 오이는 객체를 출력한다.

new를 제대로 선언하지 않으면 변수에 함수가 저장된다.

console로 찍으면 포도는 undefined를 출력한다.

객체 안의 this 는 기본적으로 window이고 엄격모드(함수 안의 맨 위에다가 'use strict' 입력-함수에만 엄격모드가 적용됨.)에서는 undefined다.

즉, 포도는 윈도우의 속성을 바꾸는 함수가 된 것이다.
new를 잊지 말자.

🌁 this의 의미

제대로 new를 선언했으면 this는 객체를 의미한다. 

                    
                      function 엉덩이(좌궁둥, 우궁둥) {
                        this.left = 좌궁둥;
                        this.right = 우궁둥;
                      }
        
                      var 오이 = new 엉덩이("조지", "마를린"); // 객체가 된 오이
        
                      console.log(오이.left); // 조지
                      console.log(오이.right); // 마를린
