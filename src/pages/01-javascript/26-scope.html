😐 스코프 😐

수정 날짜: 2021년 03월 17일

😇 개요

자바스크립트에서 무조건 알아야 하는 스코프, 실행콘텍스트, 프로토타입, 비동기, 클로저중에 스코프를 공부해보자.

💘 정의

함수 내부에 선언한 변수는 절대 함수 밖으로 빠져나가지 못한다.
이렇게 변수 및 함수의 유효범위를 스코프라고 한다. 

            
              var name = '나는 고구마';
              function log() {
                ver cucumber = '나는 오이';
              }

              console.log(cucumber);
              // 참조에러 발생
              // 함수 내부에서 선언된 변수이므로 함수 밖으로 나갈 수 없다.

💦 스코프 체인

컴퓨터는 실행하는 함수 안에 변수가 없으면 한 단계씩 위로 올라가며 변수를 찾아 나선다.

중첩된 함수에서 스코프가 헷갈릴 때, 에디터 창의 하위 함수 코드를 접으면된다.
변수가 보이지 않는다는 건 스코프로 인해 변수가 밖으로 튀어나오지 못했음을 의미한다. 그러므로 상위 함수에서 하위 함수의 변수를 이용할 수 없다.
반대로, 하위 함수에서는 상위 함수의 변수를 볼 수 있다. 이건 변수를 사용할 수 있다는 의미다.
스코프 간의 상하관계를 스코프체인이라고 부른다.
예시코드는 다음과 같다. 

            
              var name = '나는 고구마';
              var tomato = '나는 토마토';

              function log() {
                name = '나는 오이';
                var tomato = '나는 함수 내부에 선언한 변수라서 밖으로 못 나간다.';
              }

              // 전역 변수는 하위 함수에서 사용할 수 있다.
              console.log(name); // 나는 오이
              // 함수 안에 선언한 변수는 함수 밖으로 나올 수 없다.
              console.log(tomato); // 나는 토마토
            
          
            
              var name = '나는 고구마';
              var tomato = '나는 토마토';
              function log() {
                name = '나는 오이지롱';
                var tomato = '나는 함수 내부에 선언한 변수라서 밖으로 못나간다.';
              }
              // 전역 변수는 하위 함수에서 사용할 수 있다.
              console.log(name); // 나는 오이지롱
              // 함수 안에 선언한 변수는 함수 밖으로 나올presentation 수 없다.
              console.log(tomato); // 나는 토마토

💩 렉시컬 스코프 정의

자바스크립트의 변수는 선언되는 순간 저장되는 특징을 가진다.
함수 실행 위치가 아니라 선언된 위치에서 변수 찾는 것을 렉시컬 스코핑(정적 스코핑)라고 한다.

✋ 렉시컬 스코프 예시

예시1 

              
                var name = '나는 오이';

                function log() {
                 console.log(name);
                }

                function wrapper() {
                 name = '나는 고구마';
                 log();
                }

                wrapper(); // name = '나는 고구마';
  1. log 함수의 name은 언제나 전역변수인 name을 의미한다.
  2. wrapper 함수의 name은 언제나 전역변수인 name을 의미한다.
  3. wrapper을 실행하면, '나는 고구마'를 전역변수인 name에 저장된다.
  4. wrapper안의 log가 실행되면 log가 선언된 위치에서 변수를 찾는다. log는 전역변수 name에 저장되어있는 '나는 고구마'를 출력한다.
  5. wrapper(실행한 위치)에서 변수를 찾는게 아니라 선언된 위치에서 변수를 찾는다.

예시2 

              
                var name = '나는 오이';

                function log() {
                 console.log(name);
                }

                function wrapper() {
                 var name = '나는 고구마';
                 log();
                }
                wrapper(); // name = '나는 오이';
  1. log 함수의 name은 언제나 전역변수인 name을 의미한다.
  2. wrapper 함수 안에서 선언한 name은 전역변수 name과는 다른 변수이고, wrapper 밖으로 나갈 수 없다.
  3. wrapper안의 log가 실행되면, 전역변수 name에 저장된 '나는 오이'를 출력한다.

👯 렉시컬 스코프 특징

  1. 코드가 적힌 순간 스코프가 정해진다.
  2. 함수가 선언된 위치에서 변수를 찾아야 한다. 그러므로, 함수를 실행 명령에 함수를 대입해선 안 된다.
  3. 변수 뿐만 아니라 함수도 동일한 스코프를 따른다.
