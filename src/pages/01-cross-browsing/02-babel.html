🤑 개요

자바스크립트 크로스 브라우징을 도와주는 여러가지 도구 중의 하나, babel을 알아봅시다.

🏋️‍ 정의

babel
자바스크립트 컴파일러(transcompiler)
하위 호환성을 맞추기 위한 도구
매년 자바스크립트에는 새로운 스펙과 기능이 추가된다.
최신 문법을 이해하지 못하는 구형의 브라우저를 위해, 새로운 문법을 기존의 문법으로 변환해주는 역할을 한다.
지원하는 최신 문법
  1. 화살표 함수
  2. classes
  3. enhanced object literals
  4. 템플릿 스트링스
  5. destructuring
  6. let & const

📌 설치

(참고) 아래의 명령어는 프롬프트나 작업 폴더의 git bash에 다음의 명령어를 입력한다.

  1. node.js 설치

  2. npm init -y
    npm 초기 설치를 의미
    작업 폴더에 package.json 파일이 생성
  3. npm install --save-dev @babel/cli
    컴파일을 담당하는 패키지
    '--save-dev': 로컬(작업 폴더)에 설치
    구 버전: babel-cli
    최신 버전: @babel/cli
    package.json 파일 devDependencies에 내역이 남는다

  4. npm install --save-dev @babel/core

  5. npm install --save-dev @babel/preset-env
    es5 문법으로 바꿔주는 패키지
  6. npm install --save-dev babel-preset-minify
    파일 크기를 압축 패키지
    선택사항
  7. .babelrc 파일 생성
    작업 폴더에 생성
                        
                      {
                        "presets": [],
                      }
    ["minify", {builtIns: false, evaluate: false, mangle: false}],
    ["@babel/preset-env", {"targets": ">= 0.25%, not dead", "useBuiltIns":"entry", "corejs":3}],
  8. package.json 파일 수정 
                        
                      "scripts" : {},
    "build": "babel ./assets/src -d ./assets/lib -w"
    '-d' 왼쪽 경로: 원본 파일/폴더 경로
    '-d' 오른쪽 경로: es5코드로 변형된 코드가 저장될 경로
  9. npm run build
    프롬프트 또는 git bash에 입력
    babel 컴파일 실행됨
    명령 취소: ctrl + c
  10. .gitignore 파일 생성
    node_modules/ 입력

🦄 에러

Babel 설정하면 자동으로 엄격모드가 된다. 문법을 제대로 지키지 않아서 발생한 오류가 많았다. 흑흑

변수 에러
변수 선언 방식(const, let, var 예약어)을 지키지 않은 경우
for 문의 괄호 안에 변수 선언 방식을 지키지 않은 경우

ex- for (key in object) {}
for (const key in object) {}로 변경한다.
