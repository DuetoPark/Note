🙇 Gulp 🙇

👧 개요

크로스 브라우징을 돕는 많은 도구들이 존재한다.
그 중 하나인 gulp 사용법에 대해 알아봅시다.

👿 gulp란?

타입스크립트로 코드를 작성하면, 브라우저는 코드를 해석하지 못한다. 그러므로, 자바스크립트로 컴파일(컴퓨터 언어로 번역) 해야한다. 또한 html을 pug로 작성하고, CSS를 scss로 작성한다면 똑같이 컴파일 해야한다.

이런 컴파일을 해소해 주는 것을 빌드 시스템(Build system)이라 한다. Gulp는 그 많은 빌드 시스템 중 하나이다.

📌 설치

(참고) 아래의 명령어는 프롬프트나 작업 폴더의 git bash에 다음의 명령어를 입력한다.

(참고) 전역 설치 경로 C://사용자/win10/AppData/Roaming/npm/node-modules(자동으로 됨)

  1. node.js 설치

  2. npm init -y
    npm 초기 설치를 의미
    작업 폴더에 package.json 파일이 생성
  3. npm install gulp -g
    전역 폴더에 gulp 설치
    mac이면 명령어 앞에 'sudo'를 붙인다.(윈도우는 'sudo' 붙이면 에러발생)
  4. npm install --save-dev gulp
    로컬(작업 폴더)에 gulp 설치
    package.json 파일 devDependencies에 내역이 남는다.
  5. npm install --save-dev gulp-autoprefixer
    로컬(작업 폴더)에 gulp-autoprefixer 설치
    package.json 파일 devDependencies에 내역이 남는다.
    크로스 브라우징을 위한 코드(prefix)를 생성
  6. touch gulpfile.js
    gulp 설정 파일 생성
    오류발생 시: git bash에 명령어 입력 또는 직접 폴더 내에 파일 생성
  7. gulpfile.js 설정
    코드 
                          
                        var gulp = require('gulp');
                        var autoprefixer = require('gulp-autoprefixer');
  
                        gulp.task('styles', async function() {
                          gulp.src('./css/**/*.css')
                            .pipe(autoprefixer('last 2 versions'))
                            .pipe(gulp.dest('./css/build'));
                        });
    task: 실행 명령어 설정
    src: 작업 파일/폴더 경로
    dest: 폴더 생성, 변경 내용 저장

    babel 설치+동작 추가 할 예정이라면, 아래의 코드를 사용해도 상관없다.
    (단, package.json에 scripts 수정 필수)

    gulpfile.js 
                          
                        import gulp from "gulp";
                        import autoprefixer from "gulp-autoprefixer";
  
                        const styles = () =>
                          gulp
                            .src("./css/**/*.css")
                            .pipe(autoprefixer())
                            .pipe(gulp.dest("build/css"));
  
                        export const dev = gulp.series([styles]);
    package.json 
                          
                        {
                          "scripts": {
                            "dev": "gulp dev",
                          },
                        }
  8. gulp styles 또는 gulp dev
    gulp 실행
    async 오류 발생시, function 앞에 'async' 쓴다.

📌 자동갱신 (Watch)

다음의 코드를 gulpfile.js에 추가한다.

              
                gulp.task('watch', function() {
                  gulp.watch('./css/**/*.css', ['styles']);
                });

gulp.watch('실시간 작업 추적 파일 경로', ['명령어']);

babel 설치+동작 추가 할 예정이라면, 아래의 코드를 사용해도 상관없다.

              
                const live = {
                  gulp.watch('./css/**/*.css', styles);
                }
  
                export const dev = gulp.series([styles, live]);

gulp.watch('실시간 작업 추적 파일 경로', 명령어);

📌 패키지

Autoprefixer외에 지원 패키지가 다양하다.
이미지 최적화, Pug를 HTML로 변환, minify, 배포 등등

사용할 땐, 다음 목록 순서대로 진행한다.

  1. 로컬 저장(--save-dev)
  2. gulpfile.js에 import
  3. 함수 선언
  4. series 선언
