⏰ Commit 되돌리기 ⏰

작성 날짜: 2021년 06월 09일

⌚ 개요

이미 push한 commit을 취소하거나, commit은 유지하되 프로젝트를 이전 commit의 내용으로 변경해야 하는 경우가 발생한다. 이 방법에 대해 알아보자.

💡 git reset

(참고) 해당 방법으로 삭제된 커밋은 복구 불가

(참고) master 브랜치에 merge와 push를 마친 상태라면, master 브랜치에도 동일하게 적용한다.

  1. 돌아가고 싶은 commit의 아이디 확인

    Github 레포지토리 commit 히스토리에서 확인하거나, 프로젝트 파일의 Git bash에 git log를 명령한 뒤 확인한다.

  2. git checkout 브랜치명

    작업할 브랜치로 이동

  3. git reset --hard 커밋 아이디

    git log에서 커밋 아이디 이후의 모든 커밋 삭제
    --hard: add 취소, commit 취소, 작업한 내용 삭제(Modified 전 상태로 돌아감)
    파일의 작업한 내용까지 사라지므로, --hard 옵션 명령에 주의한다.

  4. git push -f origin master

    -f: (force)의 약자. 원격 저장소의 변경사항들을 강제로 덮어쓰는 옵션

  5. git log

    커밋 삭제되었는지 확인

  6. github 레포지토리에서 커밋 히스토리 확인

    로컬 저장소에도 커밋 삭제되었는지 확인

🎈 github commit 이용하기

(참고) 커밋은 유지하고 파일 내용만 되돌리고 싶은 경우에 이용

(참고) 다른 사람들의 작업을 보호할 수 있다.

  1. 깃허브 페이지에서 되돌릴 commit의 태그모양(< >) 버튼 클릭

    'Browse the repository at this point in the history' 버튼

  2. Code - Download ZIP 클릭

    초록색 버튼
    커밋 메세지를 작성할 때의 파일 내용이 저장되어 있다.

  3. 다운받은 폴더에서 필요한 파일/폴더를 프로젝트 파일에 덮어쓰기

  4. git add 파일/폴더 경로

  5. git commit -m "커밋 메세지"

  6. git push -u origin 브랜치명
