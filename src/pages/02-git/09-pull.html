🙈 Git Pull 🙈

작성 날짜: 2021년 06월 12일

🐵 개요

Github에서 Readme를 생성했는데, git log에 뜨지 않아서 30분을 헤맸다.
어이 없게도 내가 정리한 내용에 있었다.
오늘도 나는 컴퓨터하는 고릴라다. ㅠㅠ

🙊 git pull

(참고) 저장소 구분: 'github' = 원격 저장소, '우리집 컴퓨터' = 로컬 저장소

(참고) 브랜치로 구분: 'origin/master' = 원격 저장소, 'master' = 로컬 저장소

(참고) 프로젝트를 작성할 땐, 원격 저장소와 로컬 저장소의 커밋을 동일하게 맞춰야 한다.

pull을 실행하면 원격 저장소의 변경된 데이터를 가져올 수 있다.

Readme를 작성하고 수정했다는 건, 원격 저장소에서 새로운 변경이 발생했음을 의미

git pull origin master

원격 저장소의 새로운 변경사항을 로컬로 가져오기 위해 git pull 명령
